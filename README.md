# mdread

Mirror local Markdown files on a small, private web server and read them as
good-looking HTML at short URLs. Network-agnostic: binds loopback by default, with
one-flag [Tailscale](https://tailscale.com) support or a pluggable resolver for any
other network.

`mdread add notes.md` → `http://<host>:8787/<slug>`

## Features

- 📝 GitHub-styled rendering (markdown-it + highlight.js), dark mode aware
- 🧜 **Mermaid** diagrams render automatically
- 🔗 Short slugs, auto-generated from the document title
- 🗂️ Index page with **Timeline** (by day) and **By-directory** views
- ⏳ Docs auto-expire after 90 days (`--pin` to keep forever)
- 🔌 Network-agnostic binding — loopback by default, `--tailscale` for your tailnet, or any custom resolver
- ⚡ Single Bun process, no build step, vendored assets (works fully offline)

## Install

Requires [Bun](https://bun.com). [Tailscale](https://tailscale.com) is optional (only for `--tailscale` binding).

```bash
git clone https://github.com/shyamss/mdread.git
cd mdread
bun install
bun link            # exposes the `mdread` command
```

## Usage

```bash
mdread serve                 # start the server (default: 127.0.0.1:8787)
mdread serve --tailscale     # bind your Tailscale IP instead
mdread add path/to/doc.md    # publish a file; prints its URL slug
```

Then open `http://<host>:8787/`.

| Command | Description |
|---------|-------------|
| `mdread add <file.md>` | Copy + publish a file; re-adding the same path updates it |
| `mdread add --pin <file.md>` | Publish a doc that never expires |
| `mdread add --slug <name> <file.md>` | Publish at a fixed slug `/<name>` |
| `mdread list` | List published docs with expiry / pinned status |
| `mdread rm <slug>` | Remove a doc |
| `mdread prune` | Delete expired docs now |

### Config (env vars)

| Var | Default | Purpose |
|-----|---------|---------|
| `MDREAD_PORT` | `8787` | Server port |
| `MDREAD_HOST` | `127.0.0.1` | Bind host: an IP/hostname, `tailscale` (built-in resolver), or `auto` (use `MDREAD_HOST_CMD`) |
| `MDREAD_HOST_CMD` | — | Command whose output is the bind host — the pluggable hook (e.g. `ip -4 -o addr show wg0 \| grep -oP 'inet \K[\d.]+'`) |
| `MDREAD_TTL_DAYS` | `90` | Days before a doc expires |
| `XDG_DATA_HOME` | `~/.local/share` | Store root |

> **Binding precedence:** `--host` flag → `$MDREAD_HOST` → `$MDREAD_HOST_CMD` → `127.0.0.1`.
> The `tailscale`/`auto` resolvers retry at startup (the network may not be up at boot).

## How it works

- `mdread add` copies the file into `~/.local/share/mdread/docs/` and records it in
  `index.json` (atomic write). Slugs are generated by the `claude` CLI if present,
  otherwise a random id.
- The Bun server reads the store live on each request: `/` lists docs, `/<slug>`
  renders Markdown. Mermaid's runtime is vendored and served locally — no CDN.
- Host binding is pluggable (see config above); Tailscale is just one built-in resolver.

## Run as a service (Linux / systemd)

A `mdread serve` process can run as a systemd **user** service so it survives logout:

```ini
# ~/.config/systemd/user/mdread.service
[Unit]
Description=mdread markdown web server
Wants=network-online.target
After=network-online.target

[Service]
ExecStart=%h/.bun/bin/bun %h/path/to/mdread/src/mdread.ts serve --port 8787
Environment=MDREAD_HOST=tailscale     # omit to bind 127.0.0.1; or set your own
Restart=on-failure
RestartSec=5

[Install]
WantedBy=default.target
```

```bash
systemctl --user enable --now mdread
loginctl enable-linger "$USER"   # run without an active login
```

Pair it with a daily timer running `mdread prune` to clean up expired docs.

## Claude Code skill

A [Claude Code](https://claude.com/claude-code) skill is bundled in
[`skill/mdread/`](skill/mdread/SKILL.md). It teaches Claude to publish docs for you
when you say things like *"mdread that file"*.

Install it (user-level, available in every project):

```bash
mkdir -p ~/.claude/skills
cp -r skill/mdread ~/.claude/skills/        # or: ln -s "$PWD/skill/mdread" ~/.claude/skills/mdread
```

Restart Claude Code (or start a new session) and the skill is picked up
automatically. Then just say *"mdread that doc"* and Claude runs the CLI and hands
back the URL.
